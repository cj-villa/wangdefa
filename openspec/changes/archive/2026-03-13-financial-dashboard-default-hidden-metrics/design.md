## Context

理财列表页当前在进入页面时会同时展示总资产、总手续费、累计收益、昨日收益、持有产品等 dashboard 指标，列表中金额相关列也默认展开。现有实现已经具备指标卡片与列表列配置能力，本次只需要调整默认状态，不改动接口和数据计算逻辑。

## Goals / Non-Goals

**Goals:**
- 让理财列表页 dashboard 首屏默认只展示“昨日收益”。
- 让理财列表中的总金额列默认隐藏，但保留用户手动显示能力。
- 采用现有页面配置机制实现，避免引入新的状态管理复杂度。

**Non-Goals:**
- 不重构 dashboard 组件结构。
- 不修改 summary 接口返回字段。
- 不调整用户手动显示后的持久化行为边界，除非现有实现天然支持。

## Decisions

1. 复用现有 dashboard 卡片渲染结构，增加默认隐藏配置，而不是拆分为多个版本组件。
原因：这次需求只改变默认可见性，直接在现有结构上增加显隐元数据即可，变更范围最小。

2. 复用表格列配置体系处理“总金额”默认隐藏，而不是在请求层裁剪字段。
原因：列隐藏是视图行为，不应影响接口返回和表格数据完整性。

3. 默认显示策略只作用于初次进入页面的默认态。
原因：需求强调“默认隐藏”，重点是首屏体验，不应阻断已有的用户自定义显示流程。

## Risks / Trade-offs

- [默认隐藏规则与现有列持久化配置冲突] -> 优先接入现有列配置入口，避免并行维护两套默认值。
- [dashboard 组件缺少现成显隐机制] -> 在卡片定义层补充 `hiddenByDefault` 一类的元数据，保持渲染层简单。
- [字段命名与“总金额”实际列名不一致] -> 实现时以现有 columns 定义中的真实 `dataIndex` 为准，不在文档中假设具体字段名。
