## Context

理财页面最近引入了默认隐藏能力，用来降低首屏信息密度。但当前实现是直接不渲染被隐藏的指标或列，这会让布局发生变化，也不利于用户理解“这里有值但被隐藏了”。本次只调整展示层，不变更接口、不重新设计隐藏开关。

## Goals / Non-Goals

**Goals:**
- 让被隐藏的数字型内容保留原有位置并统一展示为 `**`。
- 保持现有隐藏入口和默认隐藏策略不变。
- 尽量复用现有配置，避免把脱敏逻辑散落到多个组件里。

**Non-Goals:**
- 不新增新的隐藏维度或权限体系。
- 不修改后端返回值。
- 不重做 dashboard 或列表交互结构。

## Decisions

1. 在展示层引入统一的“隐藏值格式化”逻辑，而不是修改数据源。
原因：隐藏是视图状态，不应污染接口数据或业务计算结果。

2. dashboard 和表格列都复用同一套配置或工具函数判断是否需要显示 `**`。
原因：同类隐藏行为应保持一致，避免一个区域隐藏成空白、另一个区域隐藏成占位符。

3. 隐藏时只替换数字展示，不移除标题、单位和布局容器。
原因：需求核心是“隐藏是数字展示为 `**``”，不是重新布局。

## Risks / Trade-offs

- [现有组件对 value 类型要求严格] -> 在渲染层处理为字符串展示，避免影响原始数值计算。
- [不同组件的 valueType 行为不一致] -> 优先在列 render 或指标展示层显式处理，不依赖组件隐式格式化。
- [隐藏状态来源分散] -> 抽一个共享配置或工具函数，避免后续继续分叉。
